SUMO Simulation

Simulation Environment

Network configuration based on a section of Nairobi city with 100+ intersections and 20+ routes.
Traffic demand modeling with varying vehicle types (cars, buses, trucks, emergency vehicles) and time-dependent flow patterns.
Traffic light control through TraCI (Traffic Control Interface) API for real-time signal adjustments.
Dynamic routing capabilities to adapt to changing traffic conditions and incidents.

Key Components

# SUMO configuration file (AI.sumocfg) <configuration> <input> <net-file value="nairobi.net.xml"/> <route-files value="routes.rou.xml"/> <additional-files value="detectors.add.xml"/> </input> <time> <begin value="0"/> <end value="1000"/> </time> </configuration>

Data Collection & Analysis

Vehicle trajectory data collection including position, speed, acceleration, and waiting times.
Traffic density analysis using induction loop detectors placed at strategic locations.
Performance metrics tracking: average waiting time, travel time, queue length, and emissions.
Real-time data exchange between SUMO and AI algorithms via Python scripts using TraCI.
